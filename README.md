## 5. 커밋 & 브랜치 전략

- 커밋
  - FEAT: 기능 개발
  - FIX: 기능 수정
  - BUG: 에러 수정
  - REFACTOR: 코드 전체 수정
  - DOCS: 문서 수정
- 브랜치 전략
  - main: 개발이 끝나고 dev 브랜치의 내용을 합칠 브랜치
  - dev: feat 브랜치에서 개발한 내용을 합칠 브랜치
  - feat: 기능 구현
